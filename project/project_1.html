In this project, I was fully responsible for designing, developing, and operating an end-to-end ETL (Extract, Transform, Load) pipeline that ingests data from multiple operational systems into a BigQuery-based data warehouse. The primary challenge was ensuring that data originating from various heterogeneous sources could be standardized, validated, and prepared for analytical use without compromising system performance, data consistency, or overall integrity.

This initiative was part of a broader organizational effort to modernize the data ecosystem by improving data quality, reducing architectural complexity, and delivering clean, analysis-ready datasets for business intelligence dashboards.

My Roles & Responsibilities

1. Bronze Layer (Raw Data Ingestion & Standardization)

I extracted raw data from various operational sources such as CSV and Database. At this stage, my focus was ensuring that all incoming data entered the system in a stable, complete, and structurally aligned manner. I used Talend Real-Time Big Data Platform and Google Cloud Storage/BigQuery to support and streamline the ETL workflow.

Data Extraction

  • Relational databases (MySQL, PostgreSQL, internal systems)
  • CSV files with varying sizes (small to large datasets)
  • Periodically generated exports from internal applications

Schema & Data Type Alignment

  • Standardizing column names
  • Converting data types (string → int64, float → numeric, text → timestamp)
  • Normalizing inconsistent formats
  • Fixing encoding issues
  • Standardizing delimiters and field lengths

Structural & Integrity Validation

  • Column count validation
  • Data type verification
  • Missing/null checks for critical columns
  • Duplicate detection
  • Timestamp consistency checks
  • Basic referential validation between related datasets

Loading into GCS & BigQuery

  • Google Cloud Storage (GCS) as raw staging files
  • BigQuery Bronze Tables
  • Append and Replace modes
  • Partitioning by date or timestamp
  • Clustering on frequently queried columns

The Bronze Layer ensures stable, structured datasets ready for downstream processing in the Silver Layer.

2. Silver Layer (Cleaned & Transformed Data)

In the Silver Layer, I focused on cleaning, transforming, and enriching the raw data from the Bronze Layer to make it analysis-ready. This includes standardizing formats, applying business rules, and creating derived fields for reporting and analytics. All transformations were implemented using Talend Real-Time Big Data Platform and BigQuery to ensure efficiency and scalability.

Data Transformation

  • Applying business rules and data enrichment
  • Calculating derived metrics and KPIs
  • Standardizing categorical values and formats
  • Handling nulls, missing values, and inconsistencies

Validation & Quality Checks

  • Checking referential integrity between related datasets
  • Ensuring consistency with Bronze Layer source data
  • Automated alerts for anomalies or unexpected values
  • Performance checks for query efficiency

Loading into Silver Tables

  • BigQuery Silver Tables for cleaned and transformed data
  • Partitioning by date or timestamp for query optimization
  • Clustering on frequently accessed columns
  • Incremental and full-load strategies depending on dataset

The Silver Layer ensures that data is accurate, consistent, and enriched, ready for downstream analytics, dashboards, and reporting in the Gold Layer.
